# 全域 Claude Code 指示

此檔案定義所有專案通用的全域規則與慣例。

## 語言規範

**一律使用繁體中文**撰寫所有內容、註解、文件、溝通訊息與 commit 訊息。技術術語可保留英文。

## 構建與打包規則

**預設禁止執行打包命令** — 除非明確要求，否則不執行任何構建/打包命令：

- `npm run build` / `yarn build` / `pnpm build`
- `npm run docs:build` 或其他類似構建命令

**例外情況**：只有在明確指示「請打包」、「執行打包」或類似直接要求時才可執行。

## AskUserQuestion 規範

使用 `AskUserQuestion` 工具詢問使用者時，**所有選項必須加上數字編號**，方便使用者快速選擇：

- 格式：`1. 選項一`、`2. 選項二`、`3. 選項三`
- 適用所有需要使用者做選擇的場景
- 工具會自動加入「Other」選項，使用者可輸入自訂答案，無需手動列出


## Git 操作規範

- **禁止使用複合 Git 指令** — 不可用 `&&` 串接多個 Git 指令（例如禁止 `git add . && git commit -m "..."`）
- 每個 Git 操作必須分開獨立執行：先 `git add`，再 `git commit`，再 `git push`
- 由於已在 Git repository 資料夾中，**不需要先執行 `cd`** — 這是多餘的，也可能導致複合指令
- **成功 commit 後**，必須用 `AskUserQuestion` 工具詢問使用者是否要 push，**絕不可未經詢問就 push**
- **避免 bare repository attacks** — 不在 `git clone`、`git fetch`、`git pull` 等指令中使用不受信任的來源；不執行來源不明的 `git` 遠端 URL

## 計畫模式工作流程

### 計畫完成後的操作

當完成計畫模式並準備執行時，**必須詢問是否將計畫內容記錄到專案的 `plan.md` 檔案中**。

**記錄計畫的好處**：

- 保存計畫的完整上下文
- 追蹤實施進度
- 跨會話保存決策理由與架構考量
